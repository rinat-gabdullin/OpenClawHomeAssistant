# n8n Add-on

[n8n](https://n8n.io) — мощный инструмент автоматизации рабочих процессов с открытым исходным кодом. Визуальный редактор позволяет соединять любые сервисы и создавать сложные автоматизации без написания кода.

## Запуск

После установки и запуска add-on откройте вкладку **Web UI** или перейдите по адресу `http://<HA_IP>:5678`.

## Настройки

| Параметр | Описание | По умолчанию |
|---|---|---|
| `timezone` | Временная зона | `Europe/Sofia` |
| `port` | Порт веб-интерфейса n8n | `5678` |
| `webhook_url` | Внешний URL для вебхуков (например, `https://yourdomain.com`) | — |
| `encryption_key` | Ключ шифрования для хранилища учётных данных (если пусто — генерируется автоматически) | — |
| `basic_auth_active` | Включить базовую аутентификацию | `false` |
| `basic_auth_user` | Логин для базовой аутентификации | `admin` |
| `basic_auth_password` | Пароль для базовой аутентификации | — |

## Хранение данных

Все данные (workflows, credentials, база SQLite) хранятся в `/config/.n8n` — **переживают обновления add-on'а**.

Ключ шифрования генерируется автоматически при первом запуске и сохраняется в `/config/.n8n/.encryption_key`.

## Вебхуки

Если вы хотите принимать вебхуки извне, задайте `webhook_url` в настройках — например, ваш публичный домен или DDNS.

## Примечания

- `N8N_SECURE_COOKIE` установлен в `false` — это необходимо для работы через HTTP (HA Ingress).
- По умолчанию используется SQLite. Для production рекомендуется внешняя PostgreSQL или MySQL — укажите `DB_POSTGRESDB_*` переменные через `run.sh`.
